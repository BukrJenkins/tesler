Позволяет принудительно отправить измененные поля при изменении значения в другом поле. Поле, при изменении которого будут отправляться данные задается методом setForceActive в FieldMetaBuilder
При изменении поля, помеченого ForceActive, вызывается api api/v1/row-meta/{$view_name}/{$bc_name}/{$bc_id}
Обработка данного запроса может быть реализована, объявив метод Crudma#preview
Для Inner сервисов предусмотрено более простое api (см. ResponseService#preview). По умолчанию, в AbstractResponseService ResponseService#preview вызывает ResponseService#update (дабы избежать дублирование кода при отсутствии разницы между сохранением и принудительным обновлением)
Стоит уточнить, что при работе api принудительного обновления изменения в базу НЕ сохраняются (код работает в read only транзакции, сохранение идёт только при последующем сохранении строки). Однако, для правильной работы пиклистов при изменении forceActivе поля, был предусмотрен ForceActive кеш (см. ForceActiveState.class)
