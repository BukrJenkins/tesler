В системе присутствуют возможность настройки переходов внутри приложения между различными экранами, между разлчиными подсистемами и переход на какой-то внешний ресурс.
 Существует 3 источника для таких переходов:
<ul>
  <li>data - ссылка приходит как одно одно из полей DTO при запросе данных для виджета</li>
  <li>row-meta - ссылка приходит в метаданных при выборе активной строки</li>
  <li>post-action - ссылка приходит как результат выполнения системной операции (создние новой записи, создание копии записи и т.д.)</li>
</ul>
В данном примере post-action реализован в для операции добавления новой записи, по выполнении которой произодйет переход на другую страницу
