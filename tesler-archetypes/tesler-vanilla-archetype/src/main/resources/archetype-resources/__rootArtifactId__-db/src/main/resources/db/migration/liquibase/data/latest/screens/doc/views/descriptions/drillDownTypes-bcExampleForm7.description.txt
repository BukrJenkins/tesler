В системе присутствуют возможность настройки переходов внутри приложения между различными экранами, между разлчиными подсистемами и переход на какой-то внешний ресурс.
Тип перехода задается в метаданных виджета свойствами  "drillDown": true, которое говорит о том, что по этому полю будет происходить переход, drillDownTypeKey, который хранит ключ поля,
в котором хранится тип перехода и  drillDownKey, который хранит ключ поля, которое содержит адрес перехода. В системе существует 5 типов переходов:
<ul>
  <li>Дриллдаун - переход внутри системы</li>
  <li>Относительный - относительный путь для перехода внутри одного домена между разными приложениями</li>
  <li>Относительный с новым окном - тоже самое, что относительный, но ссылка открывается в новом окне</li>
  <li>Абсолютный - содержит абсолютный путь перехода как во внутри приложения, так и на внешние ресурсы</li>
  <li>Абсолютный с новым окном - тоже самое, что и абсолютный, только открывается в новом окне</li>
</ul>
