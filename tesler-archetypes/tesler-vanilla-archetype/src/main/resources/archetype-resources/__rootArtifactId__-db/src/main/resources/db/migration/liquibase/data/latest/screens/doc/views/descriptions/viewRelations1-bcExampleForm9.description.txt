При изменении активной записи в List-виджете, система поддерживает каскадное обновление данных в дочерних виджетах, расположенных на разных View.<br>
При переходе на дочернюю View данные будут отображаться с учетом активной записи на родительской View.<br>
Связь данных на виджетах осуществляется за счет связанных БК, на котором основаны виджеты. БК виджета на дочерней View должна содержать в поле BC.PARENT_NAME имя БК родителя, использумой на основной View.<br>
Основное отличие заключается в том, что виджеты привязаны к разным View в таблице VIEW_WIDGETS.
